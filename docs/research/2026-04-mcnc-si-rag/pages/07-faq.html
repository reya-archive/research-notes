FAQ

발표 중/후 결정권자로부터 예상되는 질문과 MCNC 기술팀의 표준 답변입니다.

Q1. 왜 Claude Team은 후보에서 제외했나?

Claude Team은 Anthropic이 제공하는 소규모 팀용 플랜으로, RAG 구성이 Projects 내 파일 업로드 중심의 수동 워크플로우입니다. MCNC 요구사항인 폴더 구조 보존, 대용량 데이터셋 자동 인덱싱, 사용자별 권한 모델, SI 프로젝트 반복 배포에 최적화되어 있지 않습니다.

또한 Claude Team은 사내 시스템 커넥터가 없어 고객사 기존 시스템 연동이 제한적입니다. 엔터프라이즈 RAG 플랫폼이 아닌 콘텍스트가 제한된 팀 협업 도구이므로 본 검증 범위에서 제외했습니다.

Q2. Azure OpenAI Service는 왜 별도로 비교하지 않나요?

Microsoft 365 Copilot은 본 비교에 4번째 제품으로 포함되어 있습니다. 반면 Azure OpenAI Service 단독은 RAG 플랫폼이 아니라 LLM API/플랫폼이므로, 이를 사용해 RAG를 구현하려면 Dify + Bedrock과 유사하게 별도 오케스트레이션 레이어를 구성해야 합니다.

이는 Vertex AI Search를 단독 비교하지 않고 Gemini Enterprise가 이를 번들로 제공하는 형태로 비교한 것과 같은 맥락입니다. MCNC가 이미 Bedrock 기반 셀프호스팅 노하우를 확보한 상태이므로, 같은 역할의 Azure OpenAI를 추가 비교하는 것은 중복 투자 검토에 해당합니다.

Q3. SI 프로젝트 종료 후 SM 이관은 어떻게 되나?

4개 제품 모두 SM 이관이 가능하나 난이도가 다릅니다:

  • Q Business / Gemini Enterprise: 관리형 SaaS이므로 콘솔 권한과 커넥터 설정만 고객사 또는 SM 조직에 인계하면 됩니다. 운영 부담 최소
  • Dify + Bedrock: 셀프호스팅이라 MCNC가 운영 매뉴얼 · Runbook · 백업 · 패치 계획을 작성해 인계해야 합니다. Docker/AWS 운영 경험이 있는 조직에만 이관 권장
  • M365 Copilot: 관리형 SaaS, M365 관리자 역량만 확보하면 이관 가능

공통적으로 최소 1개월 병행 운영을 통한 지식 이전을 권장하며, 월별 SM 비용은 별도 계약에서 산정합니다.

Q4. 업로드한 내부 문서가 LLM 학습에 재사용되지는 않나?

4개 제품 모두 "고객 데이터 학습 미사용"을 약관으로 명시합니다:

  • Dify + Bedrock: Dify는 셀프호스팅이라 외부 전송 없음. Bedrock은 AWS 약관에 따라 고객 데이터를 모델 학습에 사용하지 않음. 입출력 데이터도 저장하지 않음
  • Amazon Q Business: AWS 동일 약관. HIPAA BAA · SOC 2 · ISO 27001 인증
  • Gemini Enterprise: Google Cloud 엔터프라이즈 약관에 따라 고객 데이터를 모델 학습에 사용하지 않음 (기본 설정)
  • M365 Copilot: 프롬프트 · 응답 · Graph 데이터를 기반 LLM 학습에 사용하지 않음

계약 체결 시 각 벤더의 DPA(Data Processing Agreement)를 첨부해 고객사에게 서면으로 제공할 수 있습니다.

Q5. 프로젝트 종료 후 인스턴스 · 데이터 정리는?

각 제품별 정리 절차는 다음과 같습니다:

  • Dify + Bedrock: 전용 AWS 계정 또는 서브 계정을 통째로 종료. EBS 볼륨 스냅샷 삭제. Bedrock은 무상태이므로 별도 정리 불필요
  • Q Business: Application 삭제 시 인덱스 · 대화 이력 모두 영구 삭제. S3 커넥터 소스 버킷은 별도 정리
  • Gemini Enterprise: Data Store · Agent 삭제 + Google Cloud 프로젝트 폐기. Drive 폴더는 고객사 소유라 별도 정리
  • M365 Copilot: Copilot 라이선스 해제 + SharePoint 사이트 컬렉션 정리

모든 제품에서 삭제 증빙 로그를 추출해 고객사에 완료 보고 가능합니다.

Q6. 4개 중 하나를 표준으로 고정할까, 프로젝트별로 선택할까?

MCNC 기술팀 관점 권고는 "프로젝트별 선택"입니다:

  • 고객사가 이미 Google Workspace를 쓰고 파싱 품질 · UX를 최우선할 때 → Gemini Enterprise
  • 고객사가 AWS 생태계 + 40+ 시스템 연동이 필요할 때 → Q Business
  • 데이터 주권 · 저비용 · 커스터마이징이 우선일 때 → Dify + Bedrock
  • M365/SharePoint 이미 사용 중이고 Office 앱 내장 AI가 핵심이라면 → M365 Copilot

4개 모두 MCNC 내부 Docker Compose · CDK · Google Cloud 템플릿을 확보하면 프로젝트 킥오프 시 수 시간 내 선택적 배포가 가능합니다. 단, 단일 최종 권장안은 영업팀/상급자 판단 영역이며 본 자료는 판단 재료만 제공합니다.
