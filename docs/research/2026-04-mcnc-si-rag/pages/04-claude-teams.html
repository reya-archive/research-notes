Claude Teams (Projects)

Anthropic 이 직접 제공하는 관리형 SaaS 채팅 서비스입니다. 별도 RAG 스택을 구축하지 않고 Claude 의 Projects 기능(프로젝트별 지식 파일 + 커스텀 지침)을 간이 RAG 로 활용하는 구조입니다. 4개 후보 중 구축 부담이 가장 낮습니다.

영업팀 핵심 3줄 ① 서버 · DB · 네트워크 설치가 전혀 없습니다 - Anthropic 콘솔에서 Team 생성 + 시트 구매만 하면 즉시 사용 가능. ② 1인당 월 비용은 $20 (연간 결제) 또는 $25 (월간 결제), 최소 5석부터 가입. ③ Projects 기능이 RAG 역할 - 프로젝트별로 지식 파일 · 커스텀 지침 · 대화를 묶어 관리합니다.

제품 개요

구성Anthropic claude.ai 웹/앱 + Projects (프로젝트별 지식 파일 · 커스텀 지침 · 공유 대화)
제공사Anthropic
카테고리Anthropic 직접 구독 SaaS (Team / Enterprise 플랜)
라이선스시트당 월 구독 - Team Standard $20 (연간) / $25 (월간), 최소 5석
핵심 가치인프라 0 + 가장 빠른 도입 + Claude 최신 세대 자동 업데이트

아키텍처

사용자 → Claude 웹/앱 → Anthropic 서버(Projects 저장소 + Claude API) 3단계로 동작합니다. MCNC 가 운영해야 할 컴포넌트는 0개이며, 업로드한 파일은 해당 프로젝트에 귀속되어 Anthropic 이 관리합니다.

기술팀 참고 - Projects 동작 구조

   │  브라우저 https://claude.ai  ·  데스크톱 앱  ·  모바일 앱

   │     └── Shared conversations (팀원 간 공유)
   │
   ├── Project 2 (고객사 B SI)
   └── ...
   │
   ▼
[Claude Model (Sonnet 4.6 / Haiku 4.5 / Opus 4.x)]
   · 200k context 기준 프로젝트 내 지식 파일 참조
   · 답변 생성
  • Project - 지식 파일 · 커스텀 지침 · 대화 기록을 묶는 컨테이너. 프로젝트별 공유 권한 설정 가능
  • Knowledge files - 프로젝트에 업로드하는 참조 파일. Claude 가 대화마다 자동으로 참조
  • Custom instructions - 프로젝트 공통 시스템 프롬프트 역할. 고객사별 톤 · 제약 조건 명시
  • Shared conversations - 팀원 간 대화 공유. 유용한 답변을 팀 지식으로 축적

구성 요소

Anthropic 이 운영하는 관리형 SaaS 이므로 MCNC 측 설치 구성 요소는 없습니다. 대신 Projects 내부 기능을 다음처럼 활용합니다.

기능RAG 측면 역할
Projects 하나의 SI 프로젝트 = 하나의 Project. 프로젝트별 지식 파일 · 지침 · 대화를 격리해 관리
Knowledge 파일 업로드 드래그&드롭으로 PPTX · XLSX · PDF · MD · TXT 업로드. Claude 가 필요할 때 자동 참조
Custom instructions 프로젝트 공통 시스템 프롬프트. 고객사 특수 용어 · 답변 톤 · 제약 조건 사전 정의
Google Drive · GitHub 커넥터 Team 플랜 기본 제공. 외부 소스를 Project 지식으로 자동 동기화
Claude 모델 Sonnet 4.6 (기본) · Haiku 4.5 (경량) · Opus 4.x (고난도) 중 사용자가 대화마다 선택

RAG 동작 원리

별도의 청크 · 임베딩 · 검색 파이프라인을 구축하지 않습니다. 프로젝트에 업로드한 지식 파일을 Claude 가 대화마다 자동으로 참조해 답변을 만듭니다. 200k 토큰 컨텍스트 안에 들어오는 분량은 전부 "한꺼번에" 읽는 방식입니다.

기술팀 참고 - Projects RAG 특성 (기존 RAG 와의 차이)
  • 청크 · 임베딩 없음 - 파일 전체를 컨텍스트에 넣는 방식이라 파이프라인 튜닝 여지가 없음
  • 장점 - 검색 누락 · 청크 경계 깨짐이 없고, 표/서식 손실이 최소
  • 제약 - 200k context 를 초과하는 대용량 지식 기반(수천 문서)에는 부적합. 수십~수백 파일 규모까지가 현실적
  • 적합한 규모 - SI 프로젝트 한 건 분량(고객사 제안서 · 요구사항 문서 · 업계 자료 수십 건)
  • 커스텀 지침 - "답변은 [출처: 파일명] 인용 포함" 같은 규칙을 프로젝트 단에서 공통 적용 가능

주요 기능

  • AI 모델 선택
    Claude Sonnet · Haiku · Opus
  • 한 번에 읽는 문서량

  • 문서 정리 단위

  • 지원 파일

가격 (2026-04 기준)

Team 플랜

  • Standard 시트 - $20 / 사용자 / 월 (연간 결제), $25 / 사용자 / 월 (월간 결제). 최소 5석
  • Premium 시트 - $100 / 사용자 / 월 (연간 결제), $125 / 사용자 / 월 (월간 결제). Claude Code 등 추가 사용량 필요한 개발자용
  • Team 플랜에 Projects · SSO · 중앙 청구 · 커넥터 관리 · 데스크톱 엔터프라이즈 배포 모두 포함
  • 무료 트라이얼 없음. 개인 Pro 플랜($20/월)으로 Projects 기능 사전 체험 가능

Enterprise 플랜 차이점 역할 기반 접근 권한(RBAC) · SCIM · 감사 로그 · Compliance API · 커스텀 데이터 보존 · IP 허용 목록 · HIPAA 가 필요한 고객사는 Enterprise 플랜으로 업그레이드 해야 합니다. 가격은 별도 협의.

데이터 처리 · 보안

  • 데이터 상주 - Anthropic 의 미국 인프라에 저장 · 처리됩니다. 한국 상주 옵션 없음
  • 데이터 학습 미사용 - Team / Enterprise 플랜의 프롬프트 · 응답 · 업로드 파일은 기본적으로 모델 학습에 사용되지 않음
  • 네트워크 격리 - VPC 개념 없음 (공용 SaaS). 공개 인터넷 경로로 Anthropic 서버에 도달
  • 암호화 - Anthropic 측 관리 (고객 KMS 연동 옵션 없음). 전송 구간 TLS, 저장 시 암호화
  • SSO - Team 플랜부터 SAML SSO 포함. 기존 사내 IdP 연동 간편
  • 국제 보안 인증 - SOC 2 Type II · ISO 42001. HIPAA 는 Enterprise 에서만 지원
기술팀 참고 - 규제 민감 고객사 판단 기준
  • 한국 상주 필수 고객사 (금융 · 공공 · 의료 등) - Claude Teams 비권장. Open WebUI + Bedrock (도쿄) 또는 M365 Copilot (한국 저장) 권장
  • 민감 내부 문서 외부 전송 금지 고객사 - Claude Teams 비권장. Open WebUI + Bedrock 권장 (사내 서버 내부 보관)
  • Anthropic 사용 이력 없는 고객사 - 데이터 처리 방식 사전 검토 필요 (trust.anthropic.com)
  • 빠른 시연 우선 + 데이터 상주 무관 고객사 - Claude Teams 최적 (수분 내 시작 가능)

강점과 약점

강점 5

  • 인프라 0 - 서버 · DB · 업데이트 · 백업 모두 Anthropic 책임. 구축 시간 수분
  • Projects 기능으로 RAG 대체 - 파일 업로드 + 커스텀 지침만으로 즉시 활용
  • Claude 최신 세대 즉시 사용 (Sonnet 4.6 · Opus 4.x 자동 업데이트)
  • Team 플랜부터 SSO · 중앙 청구 · 커넥터 관리 포함 - 사내 IT 부담 최소
  • 200k 컨텍스트 - Bedrock 계열과 동일한 답변 품질

약점 5

  • 데이터가 미국(Anthropic) 에 저장됨 - 한국 상주 요구 고객사 비권장
  • UI 변경 · 브랜딩 불가 - Anthropic 기본 화면 고정
  • 커스텀 커넥터 개발 불가 - Google Drive · GitHub 외 확장 제한
  • MCNC AWS · Docker 노하우 재사용 불가 - Anthropic 직접 구독 모델
  • HIPAA · 감사 로그 · SCIM · 커스텀 보존 기간은 Enterprise 플랜 필요

MCNC · SI 적합도

  • 반복 배포 - 최상. 신규 SI 프로젝트마다 Project 생성 + 지식 파일 업로드 + 고객사 인원에게 시트 할당이 전부
  • SM 이관 - 최상. MCNC 가 운영할 컴포넌트가 없어 고객사에 시트 이관 후 Anthropic 이 운영 책임
  • 프로젝트 간 이식성 - 중간. 프로젝트 템플릿(커스텀 지침 · 파일 셋)은 복제 가능하나 플랫폼 종속
  • 적합 시나리오 - 단기 PoC · 빠른 프로토타입 · Claude 답변 품질 강조가 필요한 제안 · 데이터 상주 제약이 없는 고객사
