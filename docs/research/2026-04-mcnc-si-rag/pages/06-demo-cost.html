10분 시연 시나리오

제품별 핵심 플로우

  • ① Dify + Bedrock

    Dify Knowledge 에 드래그&드롭 → 자동 인덱싱 (30초~1분) → Studio 챗 질의

    시연 포인트 1) Dify Knowledge 페이지에 프로젝트 파일 여러 개 드래그&드롭 2) "Indexing → Available" 상태 전환 확인 (실시간 진행률) 3) Studio 챗봇 앱에 지식베이스 연결 후 "이 기획서의 주요 WBS 요약해줘" 질의 4) 인용 문서 출처 하이라이트 확인

  • ② Amazon Q Business

    Web Experience 접속 → 첨부 업로드 또는 커넥터 기반 질의

    시연 포인트 1) IAM Identity Center SSO 로그인 시연 2) Web Experience에 PDF 첨부 + 질의 3) 권한 필터링 시연 (다른 사용자 계정으로 접근 불가 문서 확인) 4) 커넥터 연동 사례 (Confluence 예시)

  • ③ Gemini Enterprise + Drive

    Drive 폴더 드롭 → NotebookLM 카탈로깅 → 질의 / Deep Research

    시연 포인트 1) Drive에 프로젝트 폴더 업로드 2) NotebookLM에서 소스 선택 및 요약 자동 생성 3) "화면설계서 기반 기획 초안 작성" Deep Research 실행 4) 1M 컨텍스트로 폴더 전체 단일 질의

  • ④ Microsoft 365 Copilot

    SharePoint 업로드 → Copilot Chat 질의 → Office 앱 내 Copilot 시연

    시연 포인트 1) SharePoint 사이트에 샘플 문서 업로드 (기존 SharePoint 사용 중이면 생략) 2) Copilot Chat에서 "이 프로젝트의 핵심 요구사항을 요약해줘" 질문 3) Word에서 Copilot으로 문서 요약 시연 4) PowerPoint에서 Copilot으로 슬라이드 내용 질의 시연 5) 표준 질문 5개 순회 → 답변 품질 확인

표준 질문 5개 (공통) 및 기대 답변

  • Q1. "이 프로젝트의 주요 WBS를 단계별로 요약해줘"

    기대 답변 기획·설계·개발·테스트·배포 5단계 + 각 단계 주요 산출물 3~5개 + 예상 기간. PPTX · XLSX 파싱 품질이 여기서 드러남.

  • Q2. "화면설계서의 로그인 화면 기능 목록을 표로 정리해줘"

    기대 답변 표 형식으로 "기능명 / 입력 / 검증 / 실패 처리" 행 10여 개. Document AI(Gemini)가 가장 우수.

  • Q3. "회의록에서 결정된 액션 아이템만 뽑아서 담당자별로 묶어줘"

    기대 답변 담당자별 2~5개 액션 아이템. 출처 회의록 파일명 인용. 긴 문서 여러 개 동시 이해 - 컨텍스트 윈도우 품질.

  • Q4. "이 XLSX의 3번 시트에서 매출 추이를 분석해줘"

    기대 답변 시트명 · 행렬 인식 후 추세 해석. Q Business 시드니는 작은 표만, Gemini는 대용량도 가능. Dify는 Unstructured 연동 수준에 의존.

  • Q5. "이 프로젝트와 작년 A 프로젝트의 요구사항 차이점 알려줘"

    기대 답변 두 폴더 간 교차 검색. 동일 주제 비교 · 대조 능력. 권한 경계(Q Business) · 폴더 구분(Gemini) 품질 확인.
